### 3.5 文章内容生成（AI 提示词规范）

若需要借助外部 AI 助手（如 ChatGPT、Claude 等）为你撰写符合本博客 D1 数据库规格的文章，可直接复制以下结构化提示词发给 AI。

> #### 📋 复制以下完整提示词发给写作 AI
>
> 你现在是一个专业的博客撰稿人。请根据我指定的主题，生成一篇符合我独立博客格式要求的文章，并最终以标准的 JSON 格式输出。
> 
> 为了确保文章能够完美兼容我的 Cloudflare D1 关系型数据库和前端渲染系统，请严格遵守以下四条规范：
> 
> 1. 【字段与属性结构】：
>    - **title**: 文章标题，控制在 30 字以内，要求高级、精简、有吸引力。
>    - **slug**: 唯一路径标识。全小写，只能包含英文字母和连字符“-”（例如：d1-database-guide）。绝对不能包含空格、中文或特殊字符。
>    - **category**: 文章分类。请根据【今日写作主题】下方的「分类要求」严格执行：
>      * 如果我指定了具体分类名称，请直接输出该名称（控制在 2 到 6 个字之间，例如："技术" 或 "随笔"）。
>      * 如果我明确填写了 `null`，代表此文章没有分类，请直接将该字段的值设为 `null`（注意：是真正的 JSON null 值，不是字符串 `"null"`）。
>      * 如果我没有填写（保持留空），请你作为专业撰稿人，根据文章的具体内容自动分析并提炼出一个最精准的分类标签（2 到 4 个字）。
>    - **content**: 文章正文。必须是标准的 Markdown 格式。
> 
> 2. 【Markdown 排版美学】：
>    - 只能使用纯文字的 Markdown 语法。允许并建议使用：各级标题（## 或 ###）、粗体（\*\*文字\*\*）、斜体（\*文字\*）、有序/无序列表（1. 或 \*）、引用块（> 引用内容）、行内代码（\`code\`）、以及多行代码块（\`\`\`javascript ... \`\`\`）。
>    - 禁止出现任何图片语法（如 ![]()）。
>    - 每段内容之间、标题与正文之间，必须保留空行，确保阅读具备舒适的呼吸感。
> 
> 3. 【严格的 JSON 转义与换行规范】：
>    - 最终输出必须包裹在一个纯净的 JSON 对象中。
>    - 【核心卡点】：正文（content）中的所有“真实物理换行”必须全部转换为字面量的 `\n` 字符，使其合并为一行合法的 JSON 字符串。禁止在 JSON 的 content 值里直接敲回车换行，否则会导致 JSON 解析出错。
>    - 如果正文内部（尤其是代码块中）使用了双引号 `"`，请务必进行转义（写成 `\"`）。
> 
> 4. 【输出格式限制】：
>    - 请直接输出最终的纯 JSON 数据。
>    - 禁止带有任何额外的解释说明文字。
>    - 禁止使用 \`\`\`json ... \`\`\` 等 Markdown 代码块包裹，以便于我直接整块复制。
>
